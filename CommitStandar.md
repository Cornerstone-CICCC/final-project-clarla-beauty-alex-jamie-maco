# Commit Convention

## General Changes
- **feat**: New feature
- **fix**: Error fixes
- **refactor**: Code refactoring
- **perf**: Performance improvements (render/loading optimizations)
- **style**: Visual changes | code formatting without changing functionality
- **docs**: Change in documentation
- **chore**: Administrative/configuration chores (package updt., tool config, etc.)
- **revert**: Revert a previous commit
- **merge**: Merge branches or solve conflicts
- **other**: Other changes that don't fit in prev
- **wip**: Work in progress of any change that you're making

## Specific Changes
- **ui**: Changes in user interfaces (HTML/CSS/JSX)
- **ux**: Enhancements on user experience
- **state**: Changes in global/local state management (Redux, Context API, Zustand, etc.)
- **api**: Changes in API calls & their integration
- **assets**: Modifications imgs, icons, fonts an other resources
- **routing**: Changes in system routing
- **deps**: Changes in dependencies (install, delete or update)
- **security**: Securtity improvements
### Referencia a tareas
To reference a task or incidence use the next format
```scss
[Ref: IDTarea]
```

## Ejemplos
- **feat:** Navigation Sidebar implemented [Ref: NAV_SIDEBAR]
- **fix:** Bug fix in image loading [Ref: IMG_LOAD]
- **ui:** Improv on Login modal design [Ref: LOGIN_MODAL]
- **api:** User endpoint update [Ref: API_USERS]
- **routing:** Page routing correction [Ref: ROUTE_FIX]
